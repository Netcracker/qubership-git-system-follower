# Architecture Decision Records (ADRs)

This section contains Architecture Decision Records (ADRs) for the project. An ADR is a concise document capturing an important architectural decision made along with its context and consequences.

ADRs help to document the reasons behind technical choices, track important changes over time, and communicate the reasoning to the team and other stakeholders. Each ADR has a status, such as "Draft", "Accepted", or "Rejected".

For more details on any decision, see the individual ADR files listed below.
